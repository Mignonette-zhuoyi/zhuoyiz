01 · Context

Art therapy homework happens between sessions — without a therapist. That's the whole problem.

Therapists assign art-making and reflection exercises for clients to complete alone at home. Without guidance, clients face a creative barrier. Without structure, their emotional data evaporates before the next session. And therapists have no way to tailor or track any of it remotely.

The research question: can AI coordinate a therapeutic collaboration between two people who are never online at the same time?

CH1

Clients lack guidance during art-making — creative barriers and low self-confidence lead to homework abandonment.

CH2

Therapists can't tailor homework in a structured way, or provide between-session encouragement at scale.

CH3

Homework data depends on clients' own recall. Emotional state at the time of creation is lost by the next session.

02 · System Design

Five specialized agents. One therapeutic loop. No synchronous dependency.

The core architectural decision: instead of a single general LLM, we designed five agents each owning one specific task. This makes behavior auditable, independently improvable, and safe to deploy in a clinical context.

LLM Agent pipeline — GLM-4

Art-making Agent

Listens as clients verbally describe what they're drawing. Summarizes descriptions into structured text prompts passed to the image generation pipeline. Runs entirely client-side — no therapist involvement.

Conversation Agent

Guides the post-creation reflection phase using the therapist's pre-configured dialogue principles and example questions. The therapist's professional beliefs are embedded directly in the agent's prompt template.

Summary Agent × 2

Two separate agents condense art-making and discussion data into short session summaries for the therapist. Deliberately avoids interpretation — poses questions from a novice therapist's perspective to preserve the therapist's expert judgment.

Integration Agent

Synthesizes dialogue from both phases into three therapist-focused questions — entry points for the next in-session conversation. Turns 20 minutes of homework data into a usable pre-session brief.

AIGC pipeline — ControlNet + Stable Diffusion

Clients select semantic brushes (Human, Cloud, Ocean, Tree...) — each brush maps to a semantic concept, not an artistic style
Color-coded segments drawn by the client become a semantic segmentation map, combined with verbalized text prompts
ControlNet + Stable Diffusion v1.5 generates a photorealistic co-created artwork — client controls composition through language, not drawing skill

Memory module

Therapist's custom agent configurations persist across sessions — dialogue principles, task assignments, opening messages
Client interaction data (artwork, dialogue, color segment data) stored cross-session — emotional trend visible over time
03 · Product

Two interfaces. Opposite cognitive modes. Connected through AI.

Client — Two-phase homework workflow

Phase 1 · Art-making

Draw semantic shapes on the canvas, describe them by voice. Art-making Agent summarizes and generates a co-created image. Right panel shows the Stable Diffusion output in real time — client shapes the result through language, not artistic skill.

Phase 2 · Discussion

Conversation Agent guides multi-round reflection anchored to the finished artwork. Dialogue structure follows the therapist's pre-configured principles. All conversation data auto-syncs to the therapist's dashboard.

Therapist — Configure · Review · Adapt

The therapist-facing UI is essentially a no-code agent configuration interface. Therapists write dialogue principles and example questions that directly configure the Conversation Agent's prompt template — professional expertise embedded into AI behavior without code.

Agent Customization

Create, reorder, and edit dialogue principles. Assign homework tasks. Write a personalized opening message. Changes apply to the next session immediately.

AI-compiled History

Per-client overview with usage visualization. Full homework records: original drawing, co-created artwork, art-making process, conversation log, and AI assistant summary with therapist-facing questions.

30-day field study

354 sessions

24 clients over 30 days. 266 sessions completed both phases; 88 art-making only.

151 hours

04 · Reflections

Three things this project changed about how I think about AI systems.

01

Agent specialization is a product decision, not a technical one

We could have used one LLM for everything. We didn't — because specialization makes each agent's behavior predictable, auditable, and safe to put in a clinical context. The architecture reflects the trust model, not just the task model.

02

AI customization UI is prompt engineering for non-engineers

When therapists reorder dialogue principles in the UI, they're writing LLM system prompts — they just don't know it. Designing that abstraction right is a product design problem. The interface defines what domain experts can and can't configure.

03

AI should know when not to interpret

The Summary Agent was deliberately constrained to pose questions, not make clinical interpretations. That constraint was a product decision — keeping the therapist as the expert. In high-stakes domains, defining what AI won't do is as important as what it will.

"The hardest part wasn't building the agents — it was deciding where each one's responsibility ended. In a clinical context, those boundaries aren't technical defaults. They're design choices with real consequences."

— Project retrospective
